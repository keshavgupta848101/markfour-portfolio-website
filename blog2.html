What is semantic HTML?

HTML as a language is always evolving and there are always new elements that are being talked about or added to improve semantics and usability of websites now in the HTML specification, there were several new tags that were shipped in. Now HTML semantic tags aim to make our code more meaningful and solve this problem well it's not really a problem but it makes our code more descriptive so now we could say instead this is an article and use an article tag this is an HTML semantic tag and it tells the browser a bit more about what this content is it's a bit more descriptive so let's have a look at some of the different tags the different semantic

  1. <navbar />
  2. <header/>
  3. <figure/>
  4. <footer/>
  5. <article/>
  6. <aside/>
  7. <details />
  8. <section />
  9. <nav />
  10. <main />
  11. <mark />
  12. <summary />
  13. <time />

A semantic element clearly describes its meaning to both the browser and the developer. Examples of non-semantic elements: <div/> and <span/> - Tells nothing about its content. Examples of semantic elements: <table/>, and <article/>
